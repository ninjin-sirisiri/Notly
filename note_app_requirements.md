# Notly - 要件定義書

**アプリケーション名**: Notly (Note + Daily)

## 1. プロジェクト概要

### 1.1 目的

ノートを書く習慣をつけるための、シンプルで手軽に使えるデスクトップアプリケーション

### 1.2 ターゲットユーザー

- ノートや日記を習慣化したい人
- 既存のノートアプリが複雑すぎると感じている人
- ローカルでデータを管理したい人

### 1.3 コアコンセプト

**「最小限のUIで、最大限の習慣化」**

- 余計な機能や情報を排除
- 起動したら即座にノートが書ける
- 続けることを楽しめる仕組み

---

## 2. 技術スタック

### 2.1 フロントエンド

- **Framework**: Next.js (App Router)
- **Desktop**: Electron
- **UI Library**: TailwindCSS + shadcn/ui
- **Markdown Editor**: @uiw/react-md-editor または novel

### 2.2 バックエンド/データ管理

- **ファイルシステム**: Electron fs モジュール
- **メタデータDB**: SQLite (Prisma ORM)
- **ファイル形式**: Markdown (.md)

### 2.3 その他

- **通知**: Electron Notification API
- **日付処理**: date-fns

## 3. 機能要件

### 3.1 ノート作成・編集機能

#### 3.1.1 新規ノート作成

- アプリ起動時の動作:
  1. 空白のノートエディタを表示
  2. テンプレート選択パネルを表示(スキップ可能)
  3. テンプレート選択後、内容が自動挿入
- タイトルと本文を入力
- 日付は自動で記録され、タイトル・本文とは別の場所に表示
- Markdown記法で記述可能
- 自動保存機能(数秒ごと)
- 明示的な保存ボタンも用意

#### 3.1.2 ノート編集

- 既存ノートの編集
- 編集履歴の自動保存
- リアルタイムプレビュー(オプション)

#### 3.1.3 Markdownサポート

- 基本的なMarkdown記法(見出し、リスト、リンク、コードブロックなど)
- `[[ノート名]]` 記法によるノート間リンク
  - リンクをクリックで該当ノートへ遷移
  - 存在しないノート名の場合は新規作成を促す

---

### 3.2 ノート管理機能

#### 3.2.1 フォルダ機能

- ユーザーが自由に階層構造を作成可能
- フォルダのネスト(深さ制限なし)
- フォルダ間のノート移動(ドラッグ&ドロップ)
- フォルダの作成/削除/リネーム

#### 3.2.2 タグ機能

- ノートに複数のタグを付与可能
- タグ一覧表示
- タグによるフィルタリング
- タグの自動補完

#### 3.2.3 検索機能

- ノートのタイトルと本文を対象に全文検索
- タグ検索
- フォルダ内検索

---

### 3.3 テンプレート機能

#### 3.3.1 テンプレート作成

- ユーザーが自由にテンプレートを作成可能
- テンプレート名とMarkdown本文を設定
- テンプレートの編集/削除

#### 3.3.2 デフォルトテンプレート提供

以下のテンプレートをあらかじめ用意:

- **デイリーノート**: 日記用(日付、天気、今日の出来事など)
- **ミーティングメモ**: 会議記録用
- **アイデアメモ**: 思いついたことを記録
- **TODO**: タスク管理用
- **振り返り**: 週次/月次の振り返り用

#### 3.3.3 テンプレート利用

- ノート作成時にテンプレートを選択可能
- 選択したテンプレートの内容が自動挿入

---

### 3.4 習慣化サポート機能

#### 3.4.1 通知機能

- ノート作成を促すシンプルな通知
- 通知内容:
  - メッセージ: 「ノートを書きましょう」
  - 現在のストリーク情報を含める
- 通知時間帯を設定可能(複数設定可)
- 通知のON/OFF切り替え
- 通知をクリックするとアプリが起動/フォーカス

#### 3.4.2 ストリーク(連続日数)表示

- **新規ノート作成時のみ**カウント(編集はカウントしない)
- 1日1回、初めて新規ノートを作成した時点でカウント
- ノートを連続で書いた日数をカウント
- アプリ起動時やダッシュボードで表示
- ストリークが途切れた場合はリセット

#### 3.4.3 統計情報

- カレンダービューで日別のノート作成数を表示
- 月別/週別の統計グラフ
- 総ノート数
- 最長ストリーク記録

---

### 3.5 UI/UX要件

#### 3.5.1 ミニマルUI

- 必要最低限の情報のみ表示
- 執筆に集中できるシンプルなデザイン
- フォルダツリーは折りたたみ可能

#### 3.5.2 レイアウト構成

```
┌─────────────────────────────────────────┐
│  [Logo] ストリーク: 7日  [保存] [設定]    │
├──────────┬──────────────────────────────┤
│          │  作成日: 2025/10/07           │
│          ├──────────────────────────────┤
│ テンプレ │  タイトル: _______________     │
│ ート選択 ├───────────────────────────────┤
│          │                              │
│ フォルダ │                               │
│ ツリー   │      エディタエリア            │
│          │      (Markdown)              │
│ タグ一覧 │                               │
│          │                              │
└──────────┴──────────────────────────────┘
```

**起動時の表示:**

- 左サイドバー: テンプレート選択パネルを表示(選択後は通常のフォルダツリーに切り替え)
- 右メインエリア: 空白のノートエディタ
- 日付は自動で取得し、エディタ上部に表示

#### 3.5.3 キーボードショートカット

- `Cmd/Ctrl + N`: 新規ノート
- `Cmd/Ctrl + S`: 保存
- `Cmd/Ctrl + F`: 検索
- `Cmd/Ctrl + P`: コマンドパレット(今後実装)

## 4. データ構造

### 4.1 ファイルシステム構造

```
~/Notly/
├── notes/
│   ├── folder1/
│   │   ├── note1.md
│   │   └── note2.md
│   └── note3.md
├── templates/
│   ├── daily.md
│   └── meeting.md
└── .metadata/
    └── app.db (SQLite)
```

### 4.2 データベーススキーマ (Prisma)

```prisma
// schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = "file:./app.db"
}

model Note {
  id        String   @id @default(uuid())
  title     String
  filePath  String   @map("file_path")
  folderId  String?  @map("folder_id")
  folder    Folder?  @relation(fields: [folderId], references: [id])
  tags      Tag[]
  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")

  @@map("notes")
}

model Folder {
  id        String   @id @default(uuid())
  name      String
  parentId  String?  @map("parent_id")
  parent    Folder?  @relation("FolderHierarchy", fields: [parentId], references: [id])
  children  Folder[] @relation("FolderHierarchy")
  notes     Note[]
  createdAt DateTime @default(now()) @map("created_at")

  @@map("folders")
}

model Tag {
  id    String @id @default(uuid())
  name  String @unique
  color String?
  notes Note[]

  @@map("tags")
}

model Stat {
  date      DateTime @id
  noteCount Int      @default(0) @map("note_count")

  @@map("stats")
}

model Setting {
  key   String @id
  value String

  @@map("settings")
}
```

## 5. 非機能要件

### 5.1 パフォーマンス

- アプリ起動時間: 3秒以内
- ノート保存: リアルタイム(自動保存)
- 検索速度: 1000ノート以内で1秒以内

### 5.2 データ管理

- すべてのデータはローカルに保存
- 定期的なバックアップ機能(将来実装)
- エクスポート機能(将来実装)

### 5.3 対応OS

- macOS (Apple Silicon/Intel)
- Windows 10/11
- Linux (Ubuntu推奨)
