# 今後のロードマップと機能強化計画

アプリの正式リリースに伴い、今後の機能強化、パフォーマンス改善、およびメンテナンスの計画を以下にまとめます。

## 1. パフォーマンス改善 (優先度: 高)
ユーザー体験に直結する重要な改善項目です。ノート数が増加しても快適に動作することを保証します。

- [ ] **サイドバーのリスト仮想化 (Virtualization)**
    - **概要**: ノートやフォルダが数千件になっても描画パフォーマンスを維持するため、`react-virtuoso` などを導入して表示領域のみをレンダリングするようにします。
    - **目的**: メモリ使用量の削減、スクロールの滑らかさ向上。
- [ ] **エディタのレンダリング最適化**
    - **概要**: 非常に長いドキュメントを編集する際の入力遅延を防ぐため、Tiptapエディタの再レンダリング制御やLazy Loadingを検討します。
- [ ] **データベースクエリの最適化**
    - **概要**: SQLiteのインデックスを適切に設定し、検索やソート、フィルタリング時のレイテンシを最小化します。
- [ ] **起動時間の短縮**
    - **概要**: フロントエンドのバンドルサイズ最適化や、Tauriの初期化プロセスの見直しを行い、アプリ起動から操作可能になるまでの時間を短縮します。

## 2. コア機能の強化 (優先度: 高)
「習慣化」というアプリのコンセプトを強化し、ノートアプリとしての基本性能を高める機能群です。

- [ ] **習慣化支援機能 (Gamification)**
    - **ストリーク (連続記録)**: 毎日ノートを書いた日数をカウントし、継続を可視化します。
    - **ヒートマップ**: GitHubのような活動ログ（草）を表示し、日々の積み重ねを実感できるようにします。
    - **目標設定**: 1日の執筆文字数や時間の目標を設定できる機能。
- [ ] **整理・リンク機能の高度化**
    - **タグシステム**: フォルダ階層に依存しない整理方法として、ノートにタグを付与・管理できる機能を追加します。
    - **Wikiリンク (バックリンク)**: `[[ノート名]]` の記法でノート同士をリンクし、知識のネットワークを作れるようにします。
- [ ] **検索機能のオーバーホール**
    - **全文検索**: タイトルだけでなく、ノート本文の内容も含めた高速な検索を実装します。
    - **高度なフィルタ**: 作成日、更新日、タグ、フォルダなどを組み合わせた詳細検索。

## 3. UX/UI の改善 (優先度: 中)
使い心地を向上させ、より愛着を持ってもらうための改善です。

- [ ] **テーマと外観のカスタマイズ**
    - **カスタムテーマ**: プリセットのカラーパレットを増やし、ユーザーが好みのアクセントカラーを選べるようにします。
    - **フォント設定**: エディタやUIのフォントを変更できる機能。
- [ ] **オンボーディング体験の向上**
    - **初回ガイド**: 新規ユーザー向けに、主要機能やショートカットキーを紹介するツアー機能。
- [ ] **キーボード操作の完備**
    - **ショートカット拡充**: マウスを使わずにほぼ全ての操作（フォルダ移動、設定変更など）を行えるようにします。

## 4. 将来的な拡張 (優先度: 低)
長期的な視点で検討する大規模な機能です。

- [ ] **クラウド同期 / バックアップ**
    - Google DriveやDropbox、またはS3互換ストレージへの自動バックアップ・同期機能。
- [ ] **プラグインシステム**
    - ユーザーが独自のスクリプトや機能を追加できる拡張機能の仕組み。
- [ ] **モバイルコンパニオンアプリ**
    - 外出先でも閲覧・簡易編集ができるモバイル版の開発。
